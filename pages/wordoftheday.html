Word of the Day

Python Twilio API Wordnik API Task Scheduler SMS Automation
About This Project

Purpose & Motivation

This project started with a pretty straightforward goal: I wanted to learn new words without having to remember to check a website or download another app that would clutter my phone. The solution? A simple Python script that grabs the word of the day from Wordnik and shoots it straight to my phone via SMS using Twilio. It's low-cost, lightweight, and once you set it up, it just works in the background — no daily effort required.

Implementation & Technical Considerations

Getting this working was all about making sure it was reliable and easy to debug. The script pulls data from Wordnik's API, formats it into a text-friendly message, and sends it through Twilio. But here's the thing — I knew I'd need to troubleshoot eventually, so I built in comprehensive logging from the start. Every time the script runs, it logs whether it succeeded or failed, what the APIs returned, and whether the SMS actually went through. This way, if something breaks — maybe I hit an API limit or there's a network hiccup — I can check the logs and know exactly what went wrong. It's basically like having a paper trail that tells you the full story every single day.

Flexibility & Scheduling Options

One of my favourite parts about this project is how flexible it is. You can set it up with Windows Task Scheduler to run automatically every morning at whatever time works for you, or just fire it up manually from VS Code when you feel like testing it out. The whole setup is pretty minimal — you just need API keys from Wordnik and Twilio, plus your phone number, and you're good to go. It's also incredibly cheap to run since Twilio's free tier handles way more than one message per day, and Wordnik's API limits are generous. At the end of the day, it's a small, focused tool that does exactly what it needs to do without any unnecessary complexity.
